内容 : 日記・ブログ

ログインページ	-トップページ	-書き込み画面
				-日記閲覧画面 ⇒ 編集・削除(論理)可
				
使用言語、ツール等

		vsc

	バックエンド
		java apache tomcat

	フロントエンド
		jsp css

	DB
		PostgreSQL

	フレームワーク
		sturts

画面概要設計

	ログイン画面
		登録されたユーザー・パスワードでログインできる
		⇒登録情報はDBにて管理

	トップページ
		日記作成ボタン
		直近の日記が選択できるUI
		自分・他ユーザーの日記を閲覧できる
		自分の日記は編集・削除可能

	日記作成画面
		文章、画像を投稿可(文字数制限、画像のサイズ制限等は要検討)
		HTML使用可？
		構成 : タイトル、メッセージエリア（エラーなど）、HTMLメニュー、画像添付ボタン、本文、非公開チェックボックス、投稿ボタン、戻るボタン
		
		投稿ボタン押下時、「この内容で投稿しますか？」旨のダイアログ表示
		画面を閉じるor戻るボタン押下時、内容が既に書き込まれている場合、「進行状況は失われます」旨のダイアログ表示

	日記編集画面
		基本作成画面と同じ
		作成画面に加え、投稿ボタン⇒確定ボタン　削除ボタンを追加
		元の状態とdiffをかけ、変更があった場合には戻るボタンで「進行状況は失われます」旨のダイアログ表示
		削除ボタン押下の際には、「本当に削除しますか」旨のダイアログ表示

	日記閲覧画面
		日記を読み込み、閲覧できる
		構成 : 前ボタン、次ボタン、トップページへ戻るボタン
		構成(作成者本人の場合のみ表示) : 編集、削除ボタン
	
システム概要設計
	日記本文
		何かしらのファイルとして保存。HTMLとして書くので、HTMLファイルで保存か？(cssはHTMLファイル内に直書き？)
	画像ファイル
		ファイルとして保存。画像として格納され、DBにて対応付けされる。jspでDBを読み出す為のlogicを起こし、DBにてフルパスを受け取り、その情報をフロント側に返却し、画像を表示するイメージ
	DB
		ユーザーテーブル(ユーザーid、ユーザー名、削除フラグ)
		日記テーブル(日記id、ユーザーid、本文id、画像id、非公開フラグ、削除フラグ、作成日付、最終更新日付)
		画像テーブル(日記id、画像id、画像の保存場所(フルパス))
		本文テーブル(日記id、本文id、本文の保存場所(フルパス))
